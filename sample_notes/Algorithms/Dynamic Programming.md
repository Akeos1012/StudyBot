**Dynamic Programming** – Dynamic programming is a **problem-solving technique in programming and algorithms where complex problems are broken into smaller overlapping subproblems, and their results are stored to avoid recomputation**.

- reaks problems into smaller [[Subproblems]]
- Stores results using [[Memoization]] or [[Tabulation]]
- Avoids repeated calculations for efficiency
- Improves performance in [[Algorithms]] with overlapping computations
- Common in optimization problems like shortest paths or sequence problems
- Heavily used in [[Programming]] and competitive coding

Dynamic programming is an optimization technique that solves problems efficiently by storing and reusing previously computed results.

**Example:**
- Fibonacci sequence
    - Without DP: recalculates same values repeatedly
    - With DP: stores results and reuses them
- Shortest path problems (like [[Graph]] algorithms)
- Knapsack problem

#### How System Works?

Problem → Break into Subproblems → Solve Once → Store Results → Reuse Stored Answers → Optimized Output
