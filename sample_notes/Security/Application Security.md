**Application Security** – (AppSec) A **practice of protecting software applications from security threats, vulnerabilities, and attacks throughout their development, deployment, and operation**.

#### What it focuses on?

- Protecting [[Software]] from attacks
- Fixing vulnerabilities in [[Programming]] code
- Securing user inputs and [[Data]] handling
- Preventing unauthorized access and misuse
- Ensuring safe [[Authentication]] and [[Authorization]]

___
#### How Application Security Works?

Application security is applied throughout the software lifecycle:

- **Design phase** → secure architecture planning
- **Development phase** → secure coding practices
- **Testing phase** → vulnerability detection (static & dynamic testing)
- **Deployment phase** → security configuration and controls
- **Maintenance phase** → patching and updates

It uses tools such as:

- [[SAST]]
- [[DAST]]
- [[Runtime Monitoring]]

___

### 5 Types of Application Security

#### 1. Web Application Security

Protects web apps from:

- [[SQL Injection]]
- [[Cross-Site Scripting (XSS)]]
- [[Cross-Site Request Forgery (CSRF)]]

#### 2. Cloud Application Security

Secures apps in cloud environments:

- configuration management
- shared responsibility model
- cloud security controls

#### 3. Mobile Application Security

Focuses on Android and iOS apps:

- secure storage
- runtime permissions
- device integration security

#### 4. API Security

Protects communication between systems:

- API authentication
- data validation
- microservices security

#### 5. Container Security

Secures containerized systems:

- [[Docker]] security
- [[Kubernetes]] security
- image and runtime protection

In short, Application Security is the practice of protecting software applications from vulnerabilities and attacks by implementing secure coding, testing, and protection mechanisms throughout their lifecycle.
