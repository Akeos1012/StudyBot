**[[Containerization]]** – Containerization is a **software deployment method that packages an application together with all its dependencies (libraries, runtime, and configuration) into a lightweight, isolated unit called a container**, so it can run consistently across different environments.

- Packages application + dependencies into a single Container
- Runs consistently across different [[Operating System]] environments
- Uses shared OS kernel instead of full virtual machines
- Enables fast deployment and scaling in [[Cloud Computing]]
- Commonly managed using tools like [[Docker]] and [[Kubernetes]]
- Improves portability and efficiency in [[Distributed Systems]]

Containerization is a modern software deployment approach that isolates applications while sharing the host operating system, making software portable, scalable, and efficient.

#### How System Works?

Application + Dependencies → Container Image → Runtime Engine ([[Docker]]) → Execution on Host OS → Output

In short, containerization is a software deployment technique that packages applications with their dependencies into isolated containers so they can run consistently across different systems and cloud environments.
