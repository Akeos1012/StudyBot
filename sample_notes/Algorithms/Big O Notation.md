**Big O Notation** - A **mathematical way to describe how the performance of an algorithm changes as the input size grows**, focusing on its efficiency in terms of time or space.

It does not measure exact speed, but instead describes the _scaling behavior_ of an algorithm under increasing data. Big O Notation shows how fast or slow an algorithm grows when the amount of input data increases.

- How execution time increases with input size
- How memory usage increases with input size
- Worst-case performance behavior
- Efficiency comparison between algorithms

### Common complexity classes

- O(1) → constant time
- O(log n) → logarithmic growth
- O(n) → linear growth
- O(n log n) → linear-log growth
- O(n²) → quadratic growth
- O(2ⁿ) → exponential growth

#### Big O Flow

Algorithm → Input Growth → Performance Measurement → Complexity Classification → Optimization Decision

In short, Big O Notation is a system used to evaluate how efficiently an algorithm scales by measuring how its time and space requirements grow as input size increases.
