**Bus System** – A bus system is a **communication pathway inside a computer that transfers data, instructions, and signals between the CPU, memory, and other hardware components**.

- Acts as a shared channel for data exchange inside the computer
- Connects [[CPU]], [[Memory]], and [[Input & Output]] devices
- Transfers [[Data]], addresses, and control signals
- Helps coordinate communication between hardware components
- Essential for overall [[Computer Architecture]] performance
- Works through synchronized electrical signaling paths

A bus system is the internal communication structure that allows different parts of a computer to exchange information efficiently.

---
### Types of Bus System

- **[[Data Bus]]** → carries actual data between components
- **[[Address Bus]]** → carries memory addresses for data location
- **[[Control Bus]]** → carries control signals (read, write, timing)
- **[[System Bus]]** → combines data, address, and control buses
- **[[Expansion Bus]]** → connects external devices like GPUs or cards

**Examples:**

- CPU communicating with RAM through system bus
- GPU sending processed frames to memory
- Keyboard sending input signals to CPU
- Storage devices transferring data to system memory
- Motherboard internal communication pathways

##### Simple analogy

A bus system is like:

a highway network inside a city—cars (data) travel between buildings (components) using shared roads (buses) with traffic rules (control signals).

In short, a bus system is the internal communication structure that enables fast and organized data transfer between a computer’s main components.
