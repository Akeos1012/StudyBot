**ACID Properties** – ACID Properties are a set of rules in **[[Database]] systems** that ensure data transactions are processed reliably, consistently, and safely even when failures or multiple users are involved.

ACID Properties are rules that guarantee database transactions are reliable and error-free. They define how a database maintains correctness during operations like insert, update, or delete.
### What ACID stands for?

#### A — Atomicity
A transaction is treated as a single unit: it either fully completes or does nothing.

#### C — Consistency
A transaction brings the database from one valid state to another valid state.

#### I — Isolation
Multiple transactions run independently without interfering with each other.

#### D — Durability
Once a transaction is completed, the changes are permanently saved even after system failure.

#### Where ACID is used

- [[Database Management System]]
- [[Relational Database]]
- [[SQL]]
- [[Transaction Processing System]]

#### Why it is Important?

- Prevents data corruption
- Ensures accurate financial and business transactions
- Supports multi-user database environments
- Maintains system reliability during crashes or failures

In short, ACID Properties are database rules that ensure transactions are processed safely, consistently, and reliably even in complex or failure-prone systems.
