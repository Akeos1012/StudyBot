**Access Control** – A **security mechanism that determines who is allowed to access a system, resource, or data, and what actions they are permitted to perform**.

#### How it works

Access control checks:

- **Who you are** → identity ([[Authentication]])
- **What you can do** → permissions ([[Authorization]])

Then it either:

- allows access
- or blocks access

___
### Types of access control

#### 1. Discretionary Access Control (DAC)

- Owner decides permissions
- Example: file sharing in a folder

#### 2. Mandatory Access Control (MAC)

- System enforces strict rules
- Used in government / military systems

#### 3. Role-Based Access Control (RBAC)

- Access based on role
- Example: admin, user, guest in [[Software]] systems

#### 4. Attribute-Based Access Control (ABAC)

- Uses conditions like location, device, time
#### Where it is used?

- [[Operating System]] file permissions
- [[Database]] access rules
- [[Network Security]] systems
- [[Cloud Computing]] platforms
- [[Application Security]]

#### What it protects?

- [[Data]]
- [[System Resources]]
- [[Network]] access
- [[Software]] functions
- [[Server]] environments

In short, Access Control is a security mechanism that regulates who can access systems, data, or resources and what actions they are allowed to perform.
