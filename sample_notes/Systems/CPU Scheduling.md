**CPU Scheduling** – CPU Scheduling is a **mechanism used by the operating system to decide which process gets access to the CPU and for how long**, especially when multiple programs are competing for execution.

It is an operating system mechanism that manages CPU time allocation among processes to enable multitasking and efficient system performance.

- Managed by the [[Operating System]] kernel
- Selects which process from the [[Ready Queue]] runs next
- Uses algorithms like [[First Come First Serve]], [[Round Robin]], and [[Priority Scheduling]]
- Improves [[CPU Utilization]] and system performance
- Works closely with [[Process Management]] and [[Context Switching]]
- Enables multitasking in modern systems

CPU Scheduling is a core operating system function that controls process execution order to efficiently share CPU resources among multiple tasks.

#### How System Works?
Processes → Ready Queue → CPU Scheduler → [[CPU]] Execution → [[Context Switching]] → Repeat

In short, CPU Scheduling is an operating system process that decides which task the CPU executes next to ensure efficient and fair use of system resources.
