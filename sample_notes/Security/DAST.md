**DAST** –  is a Dynamic Application Security Testing**s method that analyzes a running application from the outside to find vulnerabilities without accessing its source code**.

- Tests [[Web Application]] while it is running
- Simulates external attacks like [[SQL Injection]] and [[XSS]]
- Does not require access to [[Source Code]]
- Identifies real-time vulnerabilities in deployed systems
- Used in [[Application Security]] testing pipelines

#### What it detects?

- [[Authentication]] weaknesses
- [[Authorization]] flaws
- Input validation issues
- [[Cross-Site Scripting (XSS)]]
- [[SQL Injection]]
- misconfigured security settings

#### How it works?

1. Tool scans a running application
2. Sends malicious or unexpected inputs
3. Observes responses and behavior
4. Flags security vulnerabilities

In short, DAST (Dynamic Application Security Testing) is a security testing method that evaluates a running application from the outside to detect vulnerabilities by simulating real-world attacks.
