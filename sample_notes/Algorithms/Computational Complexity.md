**Computational Complexity** – Computational complexity is a **measure of how much time, memory, or resources an algorithm needs to run as the size of the input increases**. It helps evaluate how efficient an algorithm is.

- Measures performance of [[Algorithms]] based on input size
- Focuses on [[Time Complexity]] and [[Space Complexity]]
- Helps compare efficiency between different solutions
- Uses notations like [[Big O Notation]] (O(n), O(log n), etc.)
- Important in Programming and [[Software]] optimization
- Guides design of efficient [[Data Structure]] and algorithms

Computational complexity describes how resource usage grows as the input grows, helping developers understand algorithm efficiency.

In short, computational complexity is the study of how efficiently an algorithm uses time and memory as the input size grows.
