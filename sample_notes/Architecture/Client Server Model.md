**Client Server Model** – The Client Server Model is a **network architecture where computing tasks are divided between clients (requesters) and servers (providers)**. In this model, clients send requests for data or services, and servers process those requests and return the appropriate responses over a network.

A fundamental system design pattern used in distributed computing where resources, data, and services are centralized on servers while multiple clients access them through communication protocols such as [[HTTP]] or [[HTTPS]] over the [[Internet]].

The Client Server Model is widely used in web applications, databases, cloud services, and enterprise systems because it allows centralized control, easier maintenance, and scalable resource distribution.

#### How System Works?

Client (User Device) → Request via [[HTTP]] / [[HTTPS]] → Server → Processing → Response → Client


In short, the Client Server Model is a distributed system architecture where clients request services and servers provide them over a network, enabling centralized management and scalable communication.
