**DNS Security** – is the **set of practices, technologies, and protocols used to protect the Domain Name System (DNS) from attacks, manipulation, and unauthorized changes, ensuring users are directed to the correct websites and services**.

- [[DNS]] records
- [[IP Address]] mappings
- [[Domain Name]] integrity
- [[Network]] routing information
- user connection requests to websites

#### What threats it prevents

- [[DNS Spoofing]] (fake DNS responses)
- [[DNS Cache Poisoning]] (corrupting stored DNS data)
- [[Man-in-the-Middle Attack]]
- redirecting users to malicious websites
- phishing via fake domain resolution

#### How DNS Security works

- encrypting DNS queries (e.g., DNS over HTTPS)
- verifying DNS responses using signatures (DNSSEC)
- validating domain authenticity
- securing DNS servers from unauthorized changes
- monitoring abnormal DNS traffic

#### Simple analogy

DNS Security is like:

protecting the internet’s address book from someone secretly rewriting entries so you end up at the wrong house.

In short, DNS Security is the set of technologies and practices that protect the Domain Name System from attacks and tampering, ensuring that domain names are correctly translated into legitimate IP addresses.
